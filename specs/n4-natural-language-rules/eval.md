# N4: 自然语言配规则 — 验收标准

## 验收步骤

### 1. 意图分类
- [ ] "帮我监控销量下降的SKU" → CREATE_RULE
- [ ] "看看现在有什么规则" → MANAGE_RULE
- [ ] "上周销量怎么样" → QUERY（不变）
- [ ] "你好" → CHAT（不变）

### 2. 规则生成
```
用户："帮我监控日销量下降超过30%的SKU，每天早上9点检查"
```
- [ ] LLM 调用 get_schema 了解数据结构
- [ ] LLM 生成结构化规则 JSON（包含 name / mcp_server / prompt_template / schedule）
- [ ] prompt_template 中包含具体的 SQL 查询逻辑且引用真实存在的表和字段
- [ ] schedule 是合法的 cron 表达式

### 3. 预览与确认
- [ ] 生成规则后展示格式化预览（名称、数据源、频率、条件、级别）
- [ ] 用户回复"确认" / "是" / "好的" → 创建规则
- [ ] 用户回复"取消" / "不要" → 取消创建
- [ ] 5 分钟无回复 → 自动取消

### 4. 规则写入
- [ ] 确认后规则写入 DB（AlertRule 表）
- [ ] 规则自动注册到 APScheduler（无需重启）
- [ ] 新规则在下一个调度周期正常执行

### 5. 规则管理
```
用户："看看现在有什么规则"
```
- [ ] 返回规则列表（名称 + 数据源 + 频率 + 状态）

```
用户："关掉销量监控的规则"
```
- [ ] 正确匹配规则
- [ ] 确认后 enabled 设为 false
- [ ] 调度任务移除

```
用户："删掉那个退货率的规则"
```
- [ ] 二次确认
- [ ] 确认后从 DB 删除 + 调度任务移除

### 6. 端到端验证
```
1. 通过对话创建一条规则
2. 等待调度触发（或手动 CLI run）
3. 验证规则正确执行并产生告警
```
- [ ] 全链路走通
