# Behavioral Boundaries

## Scope Control
- ONLY implement features defined in PRD.md
- Do NOT add features, utilities, or "improvements" not in the PRD
- Do NOT refactor code beyond what the current task requires
- Do NOT change project architecture without explicit approval

## Communication
- When requirements are ambiguous -- ASK the user, do not assume
- When blocked -- update claude-progress.txt and inform the user
- When discovering a bug or issue -- document it, don't silently fix unrelated code

## Work Discipline
- Work on ONE feature at a time
- Complete current feature before starting the next
- Follow the priority order in ROADMAP.md
- If you need to deviate from the plan, explain why
